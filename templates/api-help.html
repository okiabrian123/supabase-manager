Usage Examples

Access {{.project}} tables:

Get users data:

curl -H "apikey: {{.anonymousKey}}" \
  {{.apiEndpoint}}/rest/v1/{{.project}}_users?select=*

Get posts data:

curl -H "apikey: {{.anonymousKey}}" \
  {{.apiEndpoint}}/rest/v1/{{.project}}_posts?select=*

Insert new data (requires Service Role Key):

curl -X POST \
  -H "apikey: {{.serviceRoleKey}}" \
  -H "Content-Type: application/json" \
  -H "Prefer: return=representation" \
  -d '{"email": "user@example.com", "name": "John Doe"}' \
  {{.apiEndpoint}}/rest/v1/{{.project}}_users

Browser Access Limitation

When accessing API endpoints directly in a web browser, you cannot set custom headers. This is why you see the "No API key found in request" error.

For browser-based testing, use browser developer tools to add the API key header, or use the curl commands above in your terminal.
